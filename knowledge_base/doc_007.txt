## Semantic Search with SentenceTransformers

SentenceTransformers is a Python library built upon the Transformers framework, specifically designed for generating semantically meaningful sentence embeddings. Unlike traditional word embeddings (like Word2Vec or GloVe) which represent individual words, SentenceTransformers produce fixed-size vector representations for entire sentences or documents, capturing contextual information crucial for semantic similarity comparisons. These embeddings are *dense*, meaning all elements within the vector contribute to its meaning, contrasting with sparse representations common in techniques like TF-IDF.

The core functionality relies on pre-trained transformer models fine-tuned on tasks like Natural Language Inference (NLI). The `all-MiniLM-L6-v2` model, utilized in this project, is a particularly efficient choice, offering a strong balance between performance and computational cost. It maps input text to a 384-dimensional vector space.  Within this space, semantically similar texts are positioned closer to each other, enabling effective similarity searches.  The model architecture is based on a lightweight version of BERT, making it suitable for resource-constrained environments.

A critical limitation of `all-MiniLM-L6-v2` (and many transformer-based models) is its maximum input sequence length of 256 tokens.  Tokenization, the process of breaking down text into smaller units, typically results in more tokens than words. Consequently, any text exceeding this limit will be truncated *without warning*. This truncation leads to information loss and significantly impacts the quality of the generated embedding. 

To mitigate this, Retrieval-Augmented Generation (RAG) systems employing SentenceTransformers necessitate careful document chunking.  Documents should be divided into segments ranging from 100 to 500 words. This range balances capturing sufficient context with staying within the token limit. Overlapping chunks can further improve recall by ensuring context isn’t lost at segment boundaries.

During a search operation, both the user’s query and the pre-computed embeddings of the knowledge base documents are transformed into vectors using the same SentenceTransformer model.  Cosine similarity, a measure of the angle between two vectors, is then calculated between the query vector and each document vector.  Higher cosine similarity scores indicate greater semantic relatedness. Documents are ranked based on these scores, and the top-ranked documents are retrieved as the most relevant context for answering the user’s query.

In the Streamlit application, the SentenceTransformer model is loaded only once using the `@st.cache_resource` decorator. This caching mechanism prevents the model from being reloaded with each user interaction, dramatically improving performance and reducing resource consumption.  The model remains in memory, ready to efficiently encode new queries and compare them against the existing document embeddings.
