## Semantic Search with SentenceTransformers

SentenceTransformers is a Python library built upon the Transformers framework, specifically designed for generating semantically meaningful sentence embeddings. Unlike traditional word embeddings (like Word2Vec or GloVe) which represent individual words, SentenceTransformers produce fixed-size vector representations for entire sentences or documents, capturing contextual information crucial for semantic similarity comparisons. These embeddings are *dense*, meaning each element in the vector contributes to the overall meaning, unlike sparse representations common in techniques like TF-IDF.

The core functionality relies on pre-trained transformer models fine-tuned on tasks like Natural Language Inference (NLI). The `all-MiniLM-L6-v2` model, utilized in this project, is a particularly efficient choice, offering a strong balance between performance and computational cost. It maps input text to a 384-dimensional vector space.  Within this space, semantically similar texts are positioned closer to each other, enabling effective similarity searches.

**Technical Details & Considerations:**

* **Embedding Generation:** The model utilizes a Siamese or triplet network architecture during training, learning to map similar sentences to nearby vectors and dissimilar sentences to distant vectors.
* **Cosine Similarity:**  The similarity between vectors is typically measured using cosine similarity, calculated as the dot product of the vectors divided by the product of their magnitudes. A cosine similarity of 1 indicates perfect similarity, 0 indicates orthogonality (no similarity), and -1 indicates perfect dissimilarity.
* **Token Limit & Chunking:** A critical limitation of `all-MiniLM-L6-v2` (and many transformer models) is its maximum input token length – 256 tokens in this case.  This means any text exceeding this limit will be truncated *without warning*.  Therefore, effective implementation necessitates dividing larger documents into smaller *chunks* ranging from 100-500 words. This ensures that the entire document content is represented in the embedding space, preventing information loss.  Chunking strategies should consider sentence boundaries to avoid splitting meaningful context.
* **Caching for Efficiency:**  Loading transformer models is computationally expensive. To optimize performance in interactive applications like the Streamlit deployment, the model is loaded only once using `@st.cache_resource`. This decorator caches the model in memory, preventing redundant loading on each user interaction and significantly reducing response times.
* **Vector Database Integration:** The generated embeddings are typically stored in a vector database (e.g., Chroma, Pinecone, FAISS) for efficient similarity search at scale.  These databases are optimized for nearest neighbor searches in high-dimensional spaces.
