## Evaluation and RAGAS Metrics: A Deep Dive

Evaluating Retrieval-Augmented Generation (RAG) pipelines is critical for ensuring quality and reliability. RAGAS (Retrieval Augmented Generation Assessment) provides a robust framework for this purpose, moving beyond simple accuracy metrics to assess the core components contributing to a successful RAG system. It focuses on dissecting *how* a RAG pipeline arrives at an answer, not just *if* the answer is correct.

**Core RAGAS Metrics:**

* **Faithfulness:** This metric quantifies the extent to which the generated answer is grounded in the retrieved context. It assesses whether the answer contains information *not* present in the provided source documents – effectively measuring hallucination.  Faithfulness is scored between 0.0 and 1.0, where 1.0 indicates perfect faithfulness (the answer is entirely supported by the context) and 0.0 indicates complete hallucination. Technically, faithfulness is often determined by utilizing another LLM to verify each statement in the answer against the retrieved context.  A statement is considered faithful if it can be directly attributed to the context.

* **Answer Relevancy:**  This metric evaluates how well the generated answer addresses the user’s question. A highly relevant answer directly responds to the query and provides useful information.  Relevancy scoring typically involves an LLM judging the semantic similarity between the question and the answer.  It’s important to note that relevancy doesn’t guarantee faithfulness; an answer can be relevant but still contain fabricated information.

* **Context Precision:** This metric focuses on the retrieval stage, measuring the proportion of retrieved context chunks that are actually relevant to the question. High context precision indicates that the retrieval system is effectively filtering out irrelevant information.  A low context precision score suggests the retrieval component is returning noise, potentially hindering answer quality and increasing computational cost.  This is calculated as (Number of Relevant Chunks Retrieved) / (Total Number of Chunks Retrieved).

**Implementation & Capstone Project Considerations:**

The capstone project utilizes an `eval_node` to perform real-time faithfulness checking. This node leverages RAGAS to score the generated answer’s faithfulness. A threshold of 0.7 is implemented; if the faithfulness score falls below this value, the answer is automatically regenerated. This iterative regeneration process is capped by a maximum retry count to prevent infinite loops. This demonstrates a closed-loop RAG system actively mitigating hallucination.

**Submission Requirements & Fallback Evaluation:**

For project submission, students are required to create a dataset of 5 question-answer pairs, including a defined “ground truth” answer for comparison.  Baseline RAGAS scores for each metric (Faithfulness, Answer Relevancy, and Context Precision) must be reported.  If the RAGAS library is unavailable due to environment constraints, a fallback evaluation method utilizing manual LLM-based scoring is acceptable. This involves prompting an LLM to assess faithfulness, relevancy, and context precision, providing detailed reasoning for its scores.  Consistency in prompting and evaluation criteria is paramount when employing this manual approach.
