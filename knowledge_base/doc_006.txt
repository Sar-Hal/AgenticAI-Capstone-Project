## StateGraph and TypedDict Design in LangGraph

LangGraph’s `StateGraph` architecture relies heavily on a centrally managed state, defined and enforced through `TypedDict` structures. This design pattern ensures type safety, predictable state management, and facilitates robust agent behavior. The `StateGraph` itself represents a directed graph where nodes process and modify this shared state.

A `TypedDict` is a Python type hinting feature that allows defining dictionaries with specific key-value types. In the context of LangGraph, it serves as a schema for the agent’s memory and operational context. All nodes within the `StateGraph` operate on an instance of this `TypedDict`, reading input values and writing output values. 

For example, the `CapstoneState` `TypedDict` in a typical LangGraph application includes the following fields:

*   **`question` (str):** Stores the original user query, providing context throughout the agent’s operation.
*   **`messages` (list[Annotated[str, add_messages]]):**  This field maintains the conversation history. Crucially, the `Annotated` type with the `add_messages` reducer automatically appends new messages to this list with each turn, simplifying conversation management. The `add_messages` reducer is a LangGraph utility that handles the accumulation of messages in a structured manner.
*   **`route` (str):** Indicates the current routing decision made by the routing node – typically one of "retrieve", "tool", or "skip".
*   **`retrieved` (str):** Contains the text content retrieved from a knowledge source, such as a vector database (e.g., ChromaDB).
*   **`sources` (list[str]):**  A list of identifiers corresponding to the documents from which the `retrieved` content originated.
*   **`tool_result` (str):** Stores the output generated by the execution of a tool.
*   **`answer` (str):** Holds the final generated response to the user’s query.
*   **`faithfulness` (float):** Represents a score (between 0.0 and 1.0) indicating the faithfulness of the generated answer to the retrieved context. This is often the output of an evaluation node.
*   **`eval_retries` (int):** A counter used to prevent infinite retry loops in evaluation processes.

**Key Considerations & Best Practices:**

*   **Strict Typing:** Every field that a node intends to write to *must* be declared within the `TypedDict`. Attempting to write to an undefined field will raise a `KeyError` at runtime, providing immediate feedback on state management issues. This enforces a clear contract between nodes.
*   **Schema Evolution:**  Modifying the `TypedDict` requires careful consideration. Changes can impact all nodes interacting with the state and may necessitate code updates across the `StateGraph`.
*   **Reducer Integration:** Utilizing LangGraph’s reducers (like `add_messages`) within the `TypedDict` definition streamlines common state update patterns, reducing boilerplate code and improving maintainability.
*   **State Granularity:** Design the `TypedDict` to contain only the necessary information for the agent’s operation. Overly complex states can hinder performance and increase debugging complexity.
